"Robot Battles: RPG Game Management"

Robot Battles to gra RPG, która pozwala graczom budować i zarządzać zespołem robotów, aby rywalizować z innymi zespołami w bitwach. Gra toczy się w futurystycznym świecie, w którym roboty służą do rozrywki i rywalizacji.

Schematy:

Robot:

  • ID : unikalny identyfikator dla każdego robotu
  • Name : imię (mnazwa)
  • Type : Typ robota (np. czołg, zwinny, latający)
  • Description : Opis wyglądu i możliwości robota.
  • Parts : Tablica obiektów części, z których składa się robot (powiązany ze schematem Parts)
  • Weapons : Tablica broni, z których może korzystać robot (powiązany ze schematem Weapons)
  • Upgrades : Tablica obiektów ulepszeń, które zostały zastosowane do robota (powiązany ze schematem Upgrades)
  • Health : Ilość zdrowia robota.
  • Speed : Szybkość robota.
  • Attack Power : Ilość obrażeń, jakie mogą wyrządzić ataki robota.

Parts: części

  • Name : Nazwa części.
  • Type : Rodzaj części (np. ciało, nogi, ramiona, głowa).
  • Description : Opis wyglądu i możliwości części.
  • Health : Ilość zdrowia zapewniana przez część.
  • Speed : Premia prędkości zapewniana przez część.
  • Attack Power : Premia do ataku zapewniana przez część.
  • Rarity : Rzadkość części (np. pospolita, rzadka, epicka).

Weapons: bron

  • Name : Nazwa broni.
  • Type : Rodzaj broni (np. biała, dystansowa).
  • Description : Opis wyglądu i możliwości broni.
  • Damage : Ilość obrażeń, jakie może zadać broń.
  • Range : Zasięg broni.
  • Accuracy : Dokładność broni.
  • Rarity : Rzadkość broni (np. pospolita, rzadka, epicka).

Upgrades: odnowienia

  • Name : Nazwa uaktualnienia.
  • Type : Rodzaj ulepszenia (np. zbroja, przyspieszenie, przyspieszenie obrażeń)
  • Description : Opis wyglądu i możliwości ulepszenia
  • Attribute Boost : Kwota, o jaką ulepszenie wzmacnia określony atrybut
  • Rarity : Rzadkość ulepszenia (np. pospolite, rzadkie, epickie)

Teams: zespoły

  • Name : Nazwa zespołu.
  • Robots : Tablica obiektów robotów, które tworzą zespół. (powiązany ze schematem Robots)
  • Credits : Ilość waluty w grze posiadanej przez drużynę.
  • Wins : Liczba bitew wygranych przez drużynę.
  • Losses : Liczba bitew przegranych przez drużynę.

Battles: bitwy

  • Arena : Nazwa areny, na której toczy się bitwa
  • Opponent (Team ID) : Nazwa drużyny przeciwnej (powiązany ze schematem Teams)
  • Players : Szereg robotów kontrolowanych przez graczy, które biorą udział w bitwie (powiązany ze schematem Robots)
  • Opponents : Szereg robotów kontrolowanych przez przeciwnika, które biorą udział w bitwie (powiązany ze schematem Robots)
  • Outcome : Wynik bitwy (np. zwycięstwo, porażka)
  • Rewards : Tablica nagród, które gracze otrzymują za zwycięstwo w bitwie (powiązany ze schematem Rewards)

Story:

  • Name : Nazwa rozdziału historii
  • Description : Opis narracji rozdziału
  • Battles : Tablica obiektów bitewnych, które gracz musi ukończyć, aby przejść przez rozdział (powiązany ze schematem Battles)
  • Rewards : Tablica nagród, które gracz otrzymuje za ukończenie rozdziału (powiązany ze schematem Rewards)

Rewards: wynagoroda

  • Type : rodzaj nagrody (np. waluta w grze, przedmiot)
  • Description : Opis nagrody
  • Amount : Kwota nagrody (np. jeśli nagroda jest walutą w grze, będzie to kwota otrzymanej waluty)
  • Item : Obiekt reprezentujący nagrodę za przedmiot (powiązany ze schematem Rewards Item)

Rewards Item: wynagoroda przewdmiot

  • Name : Nazwa elementu.
  • Type : Typ przedmiotu (np. część, broń, ulepszenie).
  • Rarity : Rzadkość przedmiotu (np. pospolity, rzadki, epicki).
  • Description : Opis wyglądu i możliwości przedmiotu.
  • Health : Ilość zdrowia zapewnianego przez przedmiot. Występuje tylko wtedy, gdy typem elementu jest „część”.
  • Speed : Premia do prędkości zapewniana przez przedmiot. Występuje tylko wtedy, gdy typem elementu jest „część”.
  • Attack Power : Premia do ataku zapewniana przez przedmiot. Występuje tylko wtedy, gdy typem przedmiotu jest „część” lub „broń”.
  • Damage : Ilość obrażeń, jakie może zadać broń. Występuje tylko wtedy, gdy typem przedmiotu jest „broń”.
  • Range : Zasięg broni. Występuje tylko wtedy, gdy typem przedmiotu jest „broń”.
  • Accuracy : Dokładność broni. Występuje tylko wtedy, gdy typem przedmiotu jest „broń”.
  • Attribute Boost : Kwota, o jaką ulepszenie wzmacnia określony atrybut. Występuje tylko wtedy, gdy typem przedmiotu jest „ulepszenie”

Ranking Table Schema:

  • Rank : Pozycja gracza w ogólnej tabeli liderów.
  • Username : Nazwa użytkownika gracza.
  • Team Name : Nazwa drużyny gracza (powiązany ze schematem Teams)
  • Wins : liczba bitew wygranych przez drużynę gracza.
  • Losses : Liczba bitew przegranych przez drużynę gracza.
  • Win Percentage : Procent bitew wygranych przez drużynę gracza.
  • Elo Rating : Klasyfikacja liczbowa odzwierciedlająca poziom umiejętności gracza. Ocena jest korygowana na podstawie zwycięstw i porażek gracza z innymi graczami o podobnych ocenach.
  • Last Online : Data i godzina, kiedy gracz był ostatnio widziany online.

Functions

Gra będzie zawierała menu główne, które umożliwi graczom dostęp do różnych trybów, takich jak tryb fabularny, tryb wieloosobowy i sklep z przedmiotami

Build and Customize:

  • Gracze mogą budować i dostosowywać swoje roboty, wybierając różne części, broń i ulepszenia. Przedmioty te można kupić za walutę w grze lub zdobyć w bitwach.
  • Parts : Części są podstawowymi elementami budulcowymi robota. Istnieje kilka różnych rodzajów części, w tym ciało, nogi, ręce i głowa. Każda część ma inne atrybuty, takie jak zdrowie, szybkość i siła ataku.
  • Weapons : Broń służy do atakowania innych robotów. Istnieje kilka różnych rodzajów broni, w tym broń do walki wręcz i broń dystansowa. Każda broń ma inne atrybuty, takie jak obrażenia, zasięg i celność.
  • Upgrades : Ulepszenia służą do ulepszania atrybutów robota lub jego broni. Istnieje kilka różnych rodzajów ulepszeń, w tym zbroja, wzmacniacze prędkości i wzmacniacze obrażeń. Każde ulepszenie ma inne atrybuty, takie jak ilość zdrowia lub obrażeń, które dodaje.

Manage Team:

  • Gracze będą musieli zarządzać swoją drużyną, szkoląc swoje roboty, przydzielając role i obowiązki oraz podejmując strategiczne decyzje, aby zwiększyć swoje szanse na wygranie bitwy.
  • Training : Gracze mogą trenować swoje roboty, aby poprawić ich atrybuty, takie jak zdrowie, szybkość i siła ataku. Szkolenie wymaga czasu i zasobów, ale może prowadzić do znacznej poprawy umiejętności robota
  • Roles and Responsibilities : Gracze mogą przypisywać swoim robotom różne role i obowiązki, takie jak bojownicy pierwszej linii lub jednostki wsparcia. Każda rola ma inne atrybuty i zdolności, które można wykorzystać do uzupełnienia innych członków zespołu
  • Strategic Decisions : Gracze będą musieli podejmować strategiczne decyzje przed każdą bitwą, na przykład, które roboty wysłać jako pierwsze, jaką broń wyposażyć i jakich ulepszeń użyć

Battles:

  • Gracze mogą rywalizować z innymi drużynami w bitwach toczących się na różnych arenach, z których każda ma swoje unikalne wyzwania
  • Arenas : Istnieje kilka różnych aren, każda z różnymi układami i przeszkodami. Niektóre areny mogą zawierać niebezpieczeństwa, takie jak doły lub pułapki, których gracze będą musieli unikać
  • Battle Mechanics : Bitwy toczą się w czasie rzeczywistym i wymagają od graczy użycia zdolności i broni swoich robotów, aby pokonać przeciwną drużynę. O wyniku bitwy decyduje kombinacja strategii, umiejętności i szczęścia.
  • Rewards : Gracze zdobywają nagrody za wygrane bitwy, takie jak waluta w grze, przedmioty i punkty doświadczenia.

Story Mode:

  • Gra będzie zawierała tryb fabularny, w którym gracze będą mogli śledzić narrację, która prowadzi ich przez różne bitwy i wyzwania.
  • Narrative : W trybie fabularnym drużyna gracza będzie rywalizować w różnych turniejach i bitwach z innymi drużynami. Narracja będzie napędzana dialogami i przerywnikami filmowymi
  • Challenges : Tryb fabularny będzie zawierał różne wyzwania, takie jak bitwy z bossami i unikalne układy areny, które gracze będą musieli pokonać

Multiplayer Mode:

  • Gracze mogą również rywalizować ze sobą w trybie wieloosobowym, jako drużyny lub w bitwach jeden na jednego
  • Matchmaking : Tryb wieloosobowy będzie zawierał dobieranie graczy w pary z przeciwnikami o podobnym poziomie umiejętności
  • Team Battles : W bitwach drużynowych gracze mogą łączyć się ze znajomymi lub innymi graczami, aby rywalizować z innymi drużynami. Każdy członek zespołu będzie sterował własnym robotem
  • One-on-One Battles : Bitwy: W bitwach jeden na jednego gracze będą kontrolować jednego robota i rywalizować z innymi graczami

Ranking Table:

  • Overall Leaderboard : Tabela rankingowa powinna wyświetlać ogólną tabelę wyników wszystkich graczy w grze, posortowaną według rangi.
  • Search and Filtering : Tabela rankingowa powinna umożliwiać graczom przeszukiwanie i filtrowanie tabeli liderów na podstawie różnych kryteriów, takich jak nazwa użytkownika lub nazwa drużyny.
  • Personal Ranking : Tabela rankingowa powinna wyświetlać osobisty ranking każdego gracza, aby mogli zobaczyć, gdzie zajmują miejsce w ogólnej tabeli liderów.
  • Team Rankings : Tabela rankingowa powinna również wyświetlać rankingi drużyn, aby gracze mogli zobaczyć, jak ich drużyna wypada na tle innych.
  • Win-Loss Records : Tabela rankingowa powinna wyświetlać bilans wygranych i przegranych każdego gracza, a także procent wygranych, aby dać graczom wyobrażenie o ich wynikach.
  • Elo Rating : Tabela rankingowa powinna wyświetlać ranking Elo każdego gracza, który odzwierciedla jego poziom umiejętności w oparciu o wygrane i przegrane z innymi graczami o podobnych ocenach.
  • Rewards and Bonuses : Tabela rankingowa może oferować nagrody i bonusy graczom, którzy osiągną określone rankingi lub serię zwycięstw. Na przykład gracze, którzy dotrą do pierwszej dziesiątki, mogą otrzymać specjalny przedmiot lub premię walutową w grze.
  • Notifications : Tabela rankingowa może wysyłać powiadomienia do graczy, gdy ich ranking ulegnie zmianie lub gdy osiągną nowy kamień milowy, na przykład osiągnięcie pierwszej setki.
  • Historical Data : Tabela rankingowa może również wyświetlać dane historyczne, takie jak najlepsi gracze wszechczasów lub najlepsi gracze każdego sezonu lub miesiąca. Dałoby to graczom poczucie historii i postępów w grze.
